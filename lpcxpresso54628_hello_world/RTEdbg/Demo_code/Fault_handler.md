## Cortex-M Fault Handlers (Core Dump Demo)

Even a small data-logging buffer (e.g. 1 kB) can hold information about fatal errors and what the code was doing before the error occurred. This allows for efficient 'post-mortem' analysis. Additional information can be logged, if defined by the programmer, ranging from RTOS internals to task or system stack details and important system variables. A complete data structure or buffer can be logged with a single call to `RTE_MSGN()`. Check out the following examples.

### The following fault handler examples are available
1. Code size optimized fault handler suitable for bare-metal embedded systems: **[Simple Cortex_M4-M7_fault_handler.md](../../../STM32L433/RTEdbg/Demo_code/Simple_Cortex_M4-M7_fault_handler.md)**.
2. Fault handler suitable for RTOS-based and bare-metal embedded systems: **[Cortex_M4-M7_fault_handler.md](../../../Simple_STM32H743/RTEdbg/Demo_code/Cortex_M4-M7_fault_handler.md)**.
3. Code size optimized fault handler for bare-metal systems with an ARM Cortex-M0/M0+ core: **[Exception_handler_Cortex-M0.md](https://github.com/RTEdbg/RTEcomLib_NUCLEO_C071RB_Demo/blob/master/Exception_handler_Cortex-M0.md)**.
